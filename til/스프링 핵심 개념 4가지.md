##  Spring 4대 핵심 개념 정리 (DI, IoC, PSA, AOP)

---

## 1. DI (Dependency Injection, 의존성 주입)

###  핵심
객체가 직접 다른 객체를 생성하지 않고, **외부에서 주입받는 것**

###  왜 필요한가?
- 객체 간 **결합도 감소**
- **유지보수 및 테스트 용이**

###  예시
```java
class A {
    B b = new B();
}

class A {
    B b;

    public A(B b) {
        this.b = b;
    }
}
```

###  Spring에서 사용 방법
- `@Autowired`
- **생성자 주입 방식 권장**

---

## 2. IoC (Inversion of Control, 제어의 역전)

###  핵심
객체 생성과 관리의 주체가 **개발자가 아니라 Spring 컨테이너**

###  왜 필요한가?
- 객체 **생명주기 자동 관리**
- 개발자는 **비즈니스 로직에 집중 가능**

###  개념 흐름
- 개발자 → 객체 생성 
- Spring → 객체 생성 및 관리 

###  관계
- **DI는 IoC를 구현하는 방법 중 하나**

---

## 3. PSA (Portable Service Abstraction)

###  핵심
기술이 바뀌어도 동일한 방식으로 사용할 수 있도록 **추상화**

###  왜 필요한가?
- 기술 변경 시 **코드 수정 최소화**
- **유지보수성 증가**

###  예시
- JDBC → JPA → Hibernate 변경 가능  
```java
repository.save(entity);
```

###  대표 사례
- Spring Data JPA
- `@Transactional`

---

## 4. AOP (Aspect Oriented Programming)

###  핵심
공통 기능(로깅, 보안 등)을 **따로 분리하여 관리**

###  왜 필요한가?
- **중복 코드 제거**
- 핵심 **비즈니스 로직 집중 가능**

###  사용 예시
- 로그 출력
- 권한 체크
- 트랜잭션 처리

```java
@Aspect
@Component
public class LogAspect {

    @Before("execution(* com.example..*(..))")
    public void log() {
        System.out.println("메서드 실행 전 로그");
    }
}
```

---

##  한 줄 정리
- **DI** → 의존성을 외부에서 주입  
- **IoC** → 객체 관리를 Spring이 담당  
- **PSA** → 기술 변경에 유연하게 대응  
- **AOP** → 공통 기능을 분리하여 관리
