# 인터넷 계층 – TTL (Time To Live)

## 1. 개요
TTL(Time To Live)은 인터넷 계층에서 사용되는 값으로,  
데이터 패킷이 네트워크에서 얼마나 오래 살아있을 수 있는지를 나타내는 수치이다.

→ 쉽게 말하면 “패킷의 생명 카운트(수명 제한)”이다.

---

## 2. TTL이 필요한 이유
인터넷에서는 데이터가 여러 라우터(중간 장치)를 거쳐 이동한다.  
하지만 경로에 문제가 생기면 데이터가 같은 경로를 계속 반복할 수 있다.

→ 이를 방지하기 위해 TTL이 존재한다.  
→ 패킷이 무한히 돌아다니는 것을 막는다.

---

## 3. TTL 동작 원리
1. 데이터를 보낼 때 초기 TTL 값을 설정한다. (예: 64, 128)
2. 라우터를 하나 지날 때마다 TTL 값이 1씩 감소한다.
3. TTL 값이 0이 되면 패킷은 삭제된다.
4. 이때 ICMP를 통해 오류 메시지가 전송된다.

→ 즉, TTL은 카운트다운 방식으로 동작한다.

---

## 4. TTL 특징
### 1) 8비트 값
- TTL 범위: 1 ~ 255
- 최대값: 255

### 2) Hop 기반
- 시간 기준이 아니라 라우터를 몇 번 지났는지 기준

### 3) 자동 감소
- 라우터를 지날 때마다 자동으로 감소

---

## 5. TTL 동작 예시

초기 TTL = 5

| 단계 | TTL 값 |
|------|--------|
| 출발 | 5 |
| 라우터1 | 4 |
| 라우터2 | 3 |
| 라우터3 | 2 |
| 라우터4 | 1 |
| 라우터5 | 0 → 패킷 삭제 |

→ 목적지 도착 전에 0이 되면 데이터 전송 실패

---

## 6. TTL 사용 예시
- ping: 네트워크 상태 확인
- traceroute: 데이터 이동 경로 추적
- 오류 확인: TTL 초과 시 문제 발생 알림
- DNS/캐시: 데이터 저장 시간 제한

---

## 7. 장점
- 네트워크 무한 루프 방지
- 불필요한 트래픽 감소
- 네트워크 안정성 향상

---

## 한 줄 정리
TTL은 데이터가 네트워크에서 무한히 돌아다니지 않도록 수명을 제한하는 값이다.
