# 인증/인가 구현 과정 정리

## 1. Spring Boot 프로젝트 생성

Spring Initializr를 사용하여 프로젝트를 생성하였다.

### 추가한 의존성
- Spring Web
- Spring Security
- Spring Data JPA
- Lombok

---

# 2. 패키지 구조 설계

기능별 역할 분리를 위해 패키지를 나누어 구성하였다.

```text
controller
service
repository
entity
dto
security
```

security 내부에는 추가로 다음 패키지를 구성하였다.

```text
config
jwt
filter
```

---

# 3. User 엔티티 구현

사용자 정보를 저장하기 위해 User 엔티티를 생성하였다.

### 구성 정보
- id
- username
- password
- role

JPA의 `@Entity`를 사용하여 엔티티 클래스로 설정하였다.

---

# 4. Repository 구현

`UserRepository` 인터페이스를 생성하여 사용자 정보를 조회할 수 있도록 구현하였다.

```java
findByUsername()
```

메서드를 사용하여 사용자 검색 기능을 구성하였다.

---

# 5. DTO 구현

회원가입 및 로그인 요청 데이터를 전달하기 위해 DTO를 생성하였다.

### 구현 클래스
- SignupRequest
- LoginRequest

클라이언트 요청 데이터를 안전하게 전달하는 역할을 한다.

---

# 6. AuthService 구현

회원가입 로직을 처리하기 위해 `AuthService`를 구현하였다.

### 주요 기능
- 사용자 정보 생성
- 비밀번호 암호화
- 사용자 저장

비밀번호는 `BCryptPasswordEncoder`를 사용하여 암호화하였다.

---

# 7. AuthController 구현

클라이언트 요청을 처리하기 위해 `AuthController`를 구현하였다.

### 회원가입 API

```text
/auth/signup
```

`@PostMapping`을 사용하여 회원가입 요청을 처리하였다.

---

# 8. SecurityConfig 구현

Spring Security 설정을 위해 `SecurityConfig` 클래스를 구현하였다.

### 주요 설정
- PasswordEncoder Bean 등록
- 인증 규칙 설정
- 특정 API 접근 허용

```java
.requestMatchers("/auth/**").permitAll()
```

설정을 통해 회원가입 API 접근을 허용하였다.

---

# 9. JwtProvider 구현

JWT 생성 및 검증 기능을 담당하는 `JwtProvider` 클래스를 구현하였다.

### 구현 기능
- JWT 생성
- JWT 유효성 검사

학습용 구조로 간단한 토큰 문자열을 반환하도록 구성하였다.



# 10. JwtFilter 구현

요청마다 JWT를 검사하기 위해 `JwtFilter`를 구현하였다.

`Authorization` 헤더의 JWT를 읽어오는 구조로 구현하였다.

```java
request.getHeader("Authorization")
```

를 사용하여 토큰을 가져오도록 구성하였다.

---
